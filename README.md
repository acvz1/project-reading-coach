# 项目研读助手skill

一个可复用的 Codex Skill，用于带领学习者逐步阅读软件项目、理解代码结构，并沉淀成可复盘的 Markdown 学习笔记。

这个 Skill 来源于一次完整的 Python 项目阅读过程。它适合刚学完基础语法、想通过真实项目提升代码阅读能力和项目理解能力的学习者。

## 它能做什么

- 先建立项目地图，再进入具体代码。
- 按入口、配置、主流程、业务处理、工具函数、打包发布等层次阅读项目。
- 用初学者友好的方式解释代码块、库 API、数据流和设计意图。
- 持续维护一份 Markdown 学习笔记。
- 给每个知识点添加快速索引、代码位置、简明解释、例子和易错点。
- 总结每个模块的职责，以及它和其他模块的关系。
- 提炼可迁移的工程经验，例如函数拆分、文件拆分、防御性编程、CLI 设计、配置管理等。

## 适合什么时候用

当你想让 Codex 帮你做这些事时，可以使用这个 Skill：

```text
带我一步步阅读这个项目。
我刚学完 Python 基础，帮我理解这个代码库。
我会边问边学，请帮我持续维护学习笔记。
解释这个函数在项目流程里的作用。
读完这个模块后帮我总结职责。
从这个项目里提炼脚本编写经验。
```

## 安装方式

把整个文件夹复制到你的 Codex skills 目录，例如：

```text
%USERPROFILE%\.codex\skills\project-reading-coach
```

或者放到你自己的 skills 加载目录中。

Skill 的核心文件是：

```text
SKILL.md
```

可选的 UI 元数据文件是：

```text
agents/openai.yaml
```

## 推荐调用方式

```text
使用 project-reading-coach 带我阅读这个项目。请按模块解释代码，维护一份 Markdown 学习笔记，并总结可复用的工程经验。
```

## 推荐笔记格式

这个 Skill 默认会把学习内容整理成类似结构：

```markdown
---

### 001. 知识点标题

**知识点**

这次学到了什么。

**代码位置**

相关代码块或文件上下文。

**简明解释**

用初学者容易理解的话解释。

**小例子**

必要时提供一个最小例子。

**易错点**

以后阅读或写代码时要注意什么。
```

也会维护快速索引，例如：

```markdown
## 快速索引

- ***config.py***
- [001. 配置目录如何选择](#001-配置目录如何选择)
- ***main.py***
- [012. 主循环如何工作](#012-主循环如何工作)
```

## 核心工作流

1. 扫描项目结构，识别目录和关键文件。
2. 找到程序入口和运行链路。
3. 从外层流程读到内部实现。
4. 回答用户当前问题，优先解释选中的代码块。
5. 将学习内容追加到 Markdown 笔记，避免覆盖用户手工修改。
6. 每读完一个模块，总结它的职责和边界。
7. 项目读完后，总结目录结构、运行流程、拆分思路和防御性编程经验。

## 设计原则

- 尊重学习者的节奏，允许浅层问题反复出现。
- 优先解释当前代码，不空讲理论。
- 用数据流和调用链帮助理解项目。
- 笔记要重点突出、方便复盘，避免写成流水账。
- 修改已有笔记前先读取当前内容，尽量做最小编辑。
- 保留学习者自己的总结，并在此基础上修正和提炼。
- 将具体项目经验抽象成下一个项目也能复用的方法。

## 这个 Skill 特别适合练习什么

- 项目结构阅读学习
- 配置文件与本地状态管理
- 模块职责划分
- 函数拆分与文件拆分
- 防御性编程意识
- Markdown 学习笔记沉淀
